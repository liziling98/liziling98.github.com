策略产品经理

发表于 2019-09-20 00:00:00 | 更新于: 2019-10-26 15:17:53 | 分类于 产品

推荐系统之协同过滤（ppt复习）

  1. 信息过滤基于过去记录，协同过滤需要拥有许多用户对一系列产品打分的数据记录，算法层面上可以考虑用户相似度和item相似度，对于一个用户，寻找其他和他/她相似的（评分强相关）用户，推荐这些相似用户高评分的item给这个用户
  2. 评分类型：显式评分、隐式评分
    显式评分是用户对于一个item的打分，能够更加精准地描述用户地喜好，只是收集数据可能比较麻烦，比如评分、投票、点赞等等
    隐式评分是对用户行为的观测，收集成本更低，可能是不精确的，比如页面停留时间、商品购买等等
    评分可能是数值型的（1-5星），可能是binary型的（同意/不同意），可能是其他非标准的（文字评论）
  3. 输入输出
    输入：用户对于items的评分
    输出：推荐items到用户u_1
    items被和u_1相似的用户喜欢
  4. 挑战cold start
    新的社群/新用户/新item没有任何历史数据
  5. 推荐系统：预测（基于社区内所有用户的一系列偏好记录，预测用户可能会有多喜欢一个他/她没给过评分的产品）、推荐
  6. 组成因素：用户（提供评分的人），产品（被评分），评分（用户的兴趣）

策略产品经理（比如推荐策略产品）

专业素养：从用户角度思考问题，抓住核心用户需求的能力，需要产品指标监控分析能力+推动产品优化

  1. 用户行为 –> 用户需求
    分析用户行为，从中发现所负责产品的潜在用户需求，如何分析？
    针对某一功能，满足不同用户在不同场景下的精细化需求。

  2. 针对用户需求，通过策略满足用户的个性化需求
    那么什么是策略？
    怎么满足用户的个性化需求？

  3. 对产品进行数据分析，验证产品的目标效果，及时发现潜在问题

  4. 用户行为分析
    五个要素：时间、地点、人物、交互、交互的内容
    可以将用户行为定义成各种事件，比如搜索时间，在某一时间、某台设备、做了关于xx内容的搜索，这算是一个完整的事件。
    获取用户行为数据可以通过监测埋点看用户是否下了什么订单、点击了什么按钮之类的。
    通过用户行为分析可以知道用户画像，才能知道用户在网站上的浏览、点击、购买背后能够挖掘出怎样的需求。
    有了很多用户行为数据后，可以把用户数据按小时、或者用户级别、或者事件类别拆分成一个表，用来了解比如哪些是优质用户、哪些是即将流失的用户，之类的内容。
    可以帮助企业促进日活量（让用户更加经常使用产品），加大留存（提前发现可流失用户、降低流失率），发现高价值用户变现等等。

  5. 什么是策略
    解决问题的手段，对于推荐系统产品，策略的底层技术支撑是AI算法，当用户种类和数据种类多、用户需求复杂多变的时候，策略需要被使用，它能提出更好的方案解决问题。

  6. 满足个性化需求
    系统分析用户行为数据，建立并且细化用户画像，猜测用户喜好，甚至可以考虑时间、地点、环境等各类因素

如何从0到1打造一个产品（比如微博）

对微博的分析

微博属于一个feed流（信息单元）的产品（持续更新并呈现给用户的内容叫做feed流），它有一个显著特点是内容是从上到下依次排列的，所以很大的一个问题是动态（feed）太多，但屏幕有限，该怎么办。

由此引申的有两个问题：一是应该给用户展示什么内容（取决于产品定位），二是这些内容应该如何排序（算法排序/时间排序）。时间排序法直接按照时间先后顺序进行排序，比如微博好友圈。推荐算法排序法即运用推荐算法为用户推荐他/她可能感兴趣的内容。

流程

  1. 发现问题，想出idea，并且进行市场调研，判断需求是否合理，用户规模是怎样的，还包括对标竞品的情况，开发难度，成本等等
  2. 明确需求，沟通整个团队，比如技术团队
  3. 产品需求文档，把需求按照功能模块拆分开
  4. 产品设计，解决展示什么内容，内容如何排序之类的问题
  5. 开发过程中跟进项目、测试、模拟运营
  6. 推广上线
  7. 分析产品数据，了解DAU、安装量、用户使用时长等用户行为数据，保持与用户的沟通，不断优化产品
